# 情绪急救 App 开发需求文档

## 📋 项目概述

**项目名称**: 情绪急救 (EmotionAid / SoftEM)  
**项目类型**: 移动端心理健康应急干预应用  
**核心理念**: 在情绪危机时刻，通过极简感官互动和生理调节，快速稳定用户状态

---

## 🎯 市场分析

### 现有类似产品
1. **Calm / Headspace**: 侧重日常冥想，缺少危机时刻的极简快速响应
2. **Rootd**: 针对恐慌发作，但缺少多情绪类型支持
3. **Sanvello**: 综合性心理健康，但界面复杂，危机时不够直观
4. **Apple Watch 正念呼吸**: 功能单一，无情绪分类和定向干预

### 本产品差异化优势
✅ **极简主义**: 危机模式下零干扰，只保留核心交互  
✅ **30秒强制缓冲**: 避免用户在应激状态下做错误决策  
✅ **多情绪类型支持**: 针对恐慌、暴怒、悲伤提供不同干预方案  
✅ **可穿戴设备集成**: 被动监测+主动响应  
✅ **高度个性化**: 用户预设声音、照片、文字内容

## 🎨 核心功能模块

### 模块1: 触发系统

#### 1.1 手动触发
**界面设计**:
- 主屏幕中央显示大型红色按钮（直径占屏幕宽度60%）
- 按钮文字: "紧急求助" / "SOS"
- 支持双击或长按0.5秒触发（防误触）
- 悬浮窗模式：任何界面都可快速呼出（可选）

**技术要点**:
```
- 前端: React Native / Flutter
- 快捷手势: 三指下滑 / 电源键连按3次
- 系统级权限: 悬浮窗显示权限
```

#### 1.2 智能自动触发
**监测指标**:
- 心率异常（静息时突然超过100bpm持续30秒）
- 心率变异性HRV骤降（压力指数）
- 呼吸频率异常（>20次/分钟）

**数据来源**:
- Apple Watch / HealthKit
- Android Wear / Google Fit
- 小米手环 / 华为手环（通过官方SDK）

**触发逻辑**:
```
IF (心率 > 阈值 AND 持续时间 > 30s) OR (HRV < 阈值) THEN
  发送震动+声音提示
  显示提示卡片: "检测到您可能需要帮助，点击进入急救模式"
  用户可选择进入或忽略（30秒自动消失）
END IF
```

**技术要点**:
```
- iOS: HealthKit集成 (HKHealthStore, HKObserverQuery)
- Android: Health Connect API
- 后台运行权限配置
- 省电模式下保持监听
```

### 模块2: 呼吸引导（30秒生理缓冲）

#### 2.1 视觉设计
**动画效果**:
- 屏幕整体变暗（不透明度90%，避免完全黑屏引起恐慌）
- 中央呼吸球：
  - 吸气（4秒）: 从直径80px → 240px，颜色从深蓝渐变至浅蓝
  - 屏息（2秒）: 保持最大尺寸，轻微脉动
  - 呼气（6秒）: 缩小至80px，颜色回归深蓝
  - 循环3次 = 约36秒
  
**视觉参数**:
```javascript
呼吸球设计:
- 初始大小: 80px
- 最大大小: 240px  
- 颜色: #1e3a8a → #60a5fa (蓝色系，避免红色刺激)
- 阴影: 柔和外发光，增强专注感
- 背景: 深灰色 #1a1a1a，减少视觉刺激
```

#### 2.2 音频引导
**语音内容**:
- 开始: "跟着这个球呼吸，只需要30秒"（温和女声/男声可选）
- 过程: 轻柔背景音（海浪声/白噪音）
- 结束: "做得很好，现在告诉我你的感受"

**技术要点**:
```
- 音频格式: MP3/AAC (兼容性)
- 文本转语音: TTS引擎（支持用户自定义录音）
- 音量: 自动调节至60%，避免惊吓
- 静音模式下振动反馈配合动画
```

#### 2.3 强制流程
- **不可跳过**: 用户无法提前退出（特殊设计）
- **异常退出**: 如用户强行退出（返回桌面），30分钟后推送提醒
- **进度条**: 底部显示圆环进度（0-100%）

### 模块4: 定向安抚干预

#### 4.1 恐慌模式 - 5-4-3-2-1 Grounding

**第一步: 安全确认语音**
- 播放用户预设音频: "这是创伤记忆的警报，不是现在的危险。你现在很安全。"
- 备选: 系统默认温和语音
- 时长: 8-10秒

**第二步: 感官接地引导**
```
界面展示（依次出现，每步15秒）:

1. 看见5样东西
   "环顾四周，说出你看到的5样东西"
   [文字输入框] 或 [语音输入]
   示例: 桌子、窗户、手机、杯子、椅子

2. 触摸4样东西  
   "触摸并说出你摸到的4样东西"
   
3. 听到3种声音
   "闭上眼睛，听到什么声音？"
   
4. 闻到2种气味
   "注意周围的气味"
   
5. 品尝1种味道
   "嘴里是什么味道？"
```

**技术实现**:
```javascript
Grounding组件:
- 步骤计时器: 每步默认15秒，可手动切换下一步
- 输入方式: 文字 / 语音识别
- 进度指示: 1/5 → 5/5
- 完成奖励: "你做到了，危机已经过去"
```

#### 4.2 暴怒模式 - 安全宣泄

**交互方案A: 雾面擦除**
```
视觉设计:
- 全屏覆盖模糊雾气效果
- 用户快速摩擦屏幕，雾气逐渐消散
- 背后显示平静图像（自然风景/用户预设）
- 配音: "用力擦，把它全部擦掉"

技术实现:
- Canvas绘制雾气纹理
- 触摸轨迹检测 (touchmove事件)
- 擦除算法: destination-out混合模式
- 进度: 擦除面积达到80%视为完成
```

**交互方案B: 虚拟挤压球**
```
视觉设计:
- 3D软球位于屏幕中央
- 用户捏合手势 → 球体变形
- 松开后回弹（物理引擎）
- 球体颜色随压力变化: 蓝→紫→红

技术实现:
- Three.js / React Three Fiber (3D渲染)
- 压力反馈: 多点触控压力检测
- 震动反馈: 挤压时设备震动
- 音效: 挤压"噗噗"声
```

**交互方案C: 虚拟呐喊**
```
- 用户对着手机大喊
- 屏幕出现音波可视化（振幅动画）
- 声音越大，屏幕震动效果越强
- 持续10秒后转入平静音乐
```

**模式选择**:
- 用户首次进入时可测试3种方案
- 后续默认使用最常用的方案
- 设置中可切换

#### 4.3 悲伤模式 - 情感联结

**第一步: 照片显示**
```
视觉设计:
- 全屏展示用户预设照片（1-3张）
- 照片类型建议:
  · 亲密关系: 朋友、家人、宠物
  · 美好回忆: 旅行、庆祝场景
  · 情感支持: 拥抱、陪伴的瞬间
  
- 照片渐入效果（fade in 2秒）
- 柔和边缘模糊（营造温暖感）
```

**第二步: 自我对话播放**
```
语音内容（用户预设录音）:
- "那个为你流泪的人，是真实的。你的感受也是真实的。"
- 或其他用户自定义话语
- 时长: 10-20秒

如未设置: 
- 显示温暖文字
- 配合舒缓音乐（钢琴/弦乐）
```

**第三步: 书写释放（可选）**
```
界面:
- "把你的感受写下来，不需要完整，只是倾诉"
- 大字体文本框
- 自动保存至私密日记
- 可选择分享给指定联系人
```

**技术实现**:
```javascript
照片管理:
- 本地加密存储（AES-256）
- 支持3-5张照片预设
- 照片权限: 相册访问
- 缓存优化: 启动时预加载

音频管理:
- 录音功能: 最长60秒
- 格式: AAC/M4A
- 试听+重录功能
- 云备份（可选）
```

## 🔐 隐私与安全

### 数据安全
1. **本地优先**: 音频、照片仅存储在本地（加密）
2. **可选云备份**: 用户明确同意后才上传
3. **匿名统计**: 情绪日志仅记录类型和时间，不含个人信息
4. **无第三方追踪**: 不集成广告SDK和分析SDK

### 权限申请
```
必需权限:
- 存储权限: 保存音频和照片
- 通知权限: 自动触发提醒

可选权限:
- 健康数据: 心率监测（明确说明用途）
- 麦克风: 录制个人语音
- 相机/相册: 设置照片
- 悬浮窗: 快速呼出（Android）
```
